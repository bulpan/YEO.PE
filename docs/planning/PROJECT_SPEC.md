# 프로젝트 기획서

# 📱 YEO.PE : 블루투스 기반 휘발성 근거리 채팅 서비스 기획서

---

## 1. 서비스 개요

**서비스명:** YEO.PE  
**핵심 개념:**  
GPS 없이 **블루투스 신호(BLE)** 를 이용해 실제 가까이 있는 사람들과  
가볍게 연결하고, 휘발성 그룹채팅을 즐길 수 있는 **근거리 SNS**  

**슬로건:**  
> “지금, 진짜 내 주변 사람들과 연결되다.”

---

## 2. 서비스 콘셉트

### 🎯 서비스 목적
- 위치기반이 아닌 **실제 근접성 기반**의 새로운 SNS 형태
- 교통수단, 카페, 행사장 등 “같은 공간에 있는 낯선 사람” 간의  
  자연스러운 소통과 정보 교류를 가능하게 함
- 개인정보 교환 없이, **익명 + 휘발성** 대화 환경 제공

### 🌍 주요 활용 시나리오
- **지하철/기차/버스** 에서 근처 사람들과 대화  
- **행사장/페스티벌/공연장** 에서 같은 관심사 사람들과 연결  
- **카페/도서관/회사 휴게실** 등 일상 공간에서 즉흥적 대화  
- **관광지/핫플레이스** 에서 여행객 간 정보 교류  

---

## 3. 주요 기능

| 구분 | 기능명 | 설명 |
|------|---------|------|
| **1** | BLE 근거리 탐색 | 주변 블루투스 활성 사용자 탐색 (닉네임 일부 마스킹) |
| **2** | 방 생성 | 주변에 방이 없을 경우 최초 생성 가능, 방 이름 지정 |
| **3** | 방 참여 | BLE 탐색 결과 기반으로 근처 사용자에게 푸시 전송 → 승인 시 연결 |
| **4** | 그룹 채팅 | 텍스트, 이미지, 이모지 전송 가능 (동영상 제외) |
| **5** | 휘발성 방 정책 | 24시간 후 자동 폐기, 거리 이탈 시 자동 퇴장 |
| **6** | 회원/비회원 모드 | 비회원은 읽기만 가능, 작성/방 생성은 로그인 필요 |
| **7** | 설정 메뉴 | 닉네임 수정, 노출여부, 로그아웃/탈퇴 등 |

## 4. 기술 아키텍처

### ⚙️ 구조 개요

| 영역 | 기술 | 역할 | 상태 |

| **Web Server** | Nginx | 리버스 프록시, SSL/TLS | ✅ 완료 |
| **Infrastructure** | Oracle Cloud Infrastructure | 서버 호스팅 | ✅ 완료 |

## 5. 사용자 흐름 (User Flow)

1️⃣ **앱 실행 → BLE 권한 요청 → 탐색 시작**  
2️⃣ **주변 사용자 표시 (닉네임 일부 마스킹)**  
3️⃣ **방 생성 or 참여 선택**  
4️⃣ **푸시 승인 후 채팅방 입장**  
5️⃣ **BLE 거리 이탈 시 자동 퇴장**  
6️⃣ **24시간 후 자동 방 폐기**

---

## 6. 정책 및 제한

| 구분 | 정책 |
|------|------|
| **회원 정책** | 이메일 / 구글 / 애플 / 카카오 / 네이버 로그인 지원 |
| **비회원 정책** | 탐색 및 열람만 가능, 글쓰기 및 방 생성 불가 |
| **데이터 보관** | 채팅 내용은 서버 TTL(24h) 후 자동 삭제 |
| **프라이버시 보호** | 닉네임 일부 마스킹, BLE UUID 임시 생성 |
| **악용 방지** | 메시지 작성은 인증된 계정만 허용 |

---

## 7. 보안 및 운영 전략

- **BLE UUID 무작위화** : 사용자의 실제 기기 식별 방지  
- **JWT 기반 세션 토큰** : 인증 및 접근 제어  
- **메시지 암호화 (AES-256)** : 서버 경유 데이터 보호  
- **로그 모니터링** : 부적절 행위 탐지 및 자동 차단  
- **서버 TTL 자동 관리** : 만료 데이터 즉시 삭제  

---

## 8. 사업성 분석

### 📈 시장 기회
- BLE 기술 상용화로 인한 **근거리 네트워킹 시장 성장**
- SNS 포화 이후 “로컬 실시간 커뮤니케이션”으로 이동 중  
  (예: AirChat, BeReal, YikYak 등 신흥 SNS 성공 트렌드)
- GPS 기반 SNS의 **정확도·프라이버시 한계 극복**  

### 💡 수익모델
1. **광고형 무료앱 + 인앱결제 광고제거 버전**
2. **향후 상업 제휴 모델**  
   (매장 BLE 비콘 연동 → 오프라인 프로모션 제공)
3. **엑싯 전략 중심의 초기 성장 모델**

## 9. 경쟁 서비스 비교

| 서비스명 | 기술 방식 | 성공 여부 | 한계점 | YEO.PE 차별점 |
|-----------|-------------|------------|----------|----------------|
| **Bridgefy** | BLE 메시징 직접 전송 | 부분 성공 (오프라인 메신저) | 연결 불안정, 속도 느림 | 서버 경유 구조로 안정성 확보 |
| **FireChat** | Wi-Fi Direct + BLE | 단기 유행 | 연결 지연, 유지 실패 | BLE 탐색 + 클라우드 채팅 결합 |
| **AirChat** | GPS 기반 근거리 SNS | 미국 중심 소규모 성장 | GPS 오차 큼, 실거리감 부족 | BLE 정밀 근접 탐색 |
| **YikYak** | GPS 기반 익명 SNS | 재런칭 후 유지 | 위치 데이터 부담 | GPS 無, BLE 탐색 중심 |

## 10. 서비스 확장 로드맵

| 단계 | 시기 | 목표 | 주요 내용 |
|------|------|------|-----------|
| **1단계 (MVP)** | 초기 | BLE 탐색 + 서버 채팅 | 기본 기능 및 휘발성 방 구현 |
| **2단계** | 런칭 3개월 내 | 사용자 유지 | 즐겨찾기, 푸시 재연결 |
| **3단계** | 6개월 | 커뮤니티화 | 주제 기반 방 (#카페, #버스, #행사장) |
| **4단계** | 1년 | 상업 연동 | BLE 비콘 상점 연동 프로모션 |
| **5단계** | 이후 | 글로벌 런칭 | 다국어 지원, 리전 서버 분산 |

## 11. 기대효과

- **GPS 없이도 현실감 있는 소셜 연결 구현**
- **즉흥적, 일시적, 익명 기반의 새로운 SNS 트렌드 선도**
- **글로벌 확장성 높은 하이브리드 통신 구조**
- **BLE 기술을 활용한 실제 “공간 중심” 커뮤니티 플랫폼 실현**

---

## 12. 결론

**YEO.PE** 는  
'BLE 탐색 + 서버 채팅'의 하이브리드 모델을 기반으로  
현실 공간에서의 즉흥적 연결과 소통을 실현하는  
차세대 휘발성 SNS 서비스다.

> "지도 밖에서 만나는 진짜 연결, YEO.PE."
